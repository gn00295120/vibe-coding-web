# Security Policy / 安全政策

## Reporting a Vulnerability / 回報安全漏洞

If you discover a security vulnerability, please report it responsibly:

如果您發現安全漏洞，請負責任地回報：

1. **DO NOT** create a public GitHub issue
   **請勿**建立公開的 GitHub issue

2. Contact us through the [Contact Page](https://vibe.lucas-futures.com/contact)
   透過[聯絡頁面](https://vibe.lucas-futures.com/contact)聯繫我們

3. Include as much detail as possible:
   請盡可能提供詳細資訊：
   - Type of vulnerability / 漏洞類型
   - Steps to reproduce / 重現步驟
   - Potential impact / 潛在影響

## Response / 回應

We will acknowledge receipt within **48 hours** and provide updates on the resolution.

我們會在 **48 小時**內確認收到，並提供解決進度更新。

## Scope / 範圍

This policy applies to:
此政策適用於：

- https://vibe.lucas-futures.com
- Related APIs and services / 相關 API 和服務

## Out of Scope / 不適用範圍

- Third-party services (Stripe, Supabase, Google OAuth)
  第三方服務
- Social engineering attacks
  社交工程攻擊
- Denial of service attacks
  阻斷服務攻擊
